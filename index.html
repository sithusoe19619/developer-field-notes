Developer Field Notes

Practical reminders for moving through bootcamp projects with Git, GitHub, and a steady daily workflow.

Git vs GitHub

Git is the local version control tool that tracks snapshots of a project on my computer. GitHub is a remote hosting service where I can store repositories online, collaborate with others, open issues, review pull requests, and deploy static sites.

Daily Loop

  1. Edit files in the working directory.
  2. Stage focused changes with git add.
  3. Commit the staged snapshot with a clear message.
  4. Push commits to GitHub with git push.

Command Reference

git status
Shows changed, staged, and untracked files.
git diff
Displays unstaged changes before they are committed.
git add
Moves selected changes into the staging area.
git commit -m
Saves staged work with a short message.
git log --oneline
Shows a compact list of recent commits.
git branch
Lists branches and shows the current branch.
git checkout -b
Creates a new branch and switches to it.
git pull
Downloads and integrates remote changes.
git push
Sends local commits to the remote repository.

Things I Should Not Commit

  • .env files because they can contain private keys, passwords, and API tokens.
  • Secrets of any kind, even if they look temporary or are only used for practice.
  • node_modules because dependencies can be reinstalled from package files.
  • .DS_Store because it is a local macOS system file, not project source code.
